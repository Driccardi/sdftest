## Period Close Process

### Month-End Close Timeline

**Day 1-2 (After Month-End)**:
- All transactions cut off at 11:59 PM on last day of month
- No backdating allowed without approval
- System generates reminder to complete transactions

**Day 3**:
- Subsidiaries run preliminary reports
- Identify missing transactions
- Post accruals and adjustments
- Reconcile bank accounts
- Reconcile credit card accounts

**Day 4**:
- Run variance analysis
- Controller reviews journal entries
- Complete intercompany reconciliations (future)
- Post any corrections

**Day 5**:
- Final review by Controller
- **Lock Period** (change status to Closed)
- Generate management reports
- Distribute reports to stakeholders

**Day 6-7**:
- Review with CFO
- Board package preparation

### Period Status Workflow

**Statuses Available**:
1. **Not Open**: Future periods, no transactions allowed
2. **Open**: Current period, all transactions allowed
3. **Closed**: Past period, locked (requires approval to reopen)

**Status Change Authorization**:
- Open Period → Closed: Controller
- Closed → Reopened: CFO approval required
- Not Open → Open: Automatic (based on date)

## Adjusting Period Configuration

**Adjusting Period Name**: Adj Jan 2026, Adj Feb 2026, etc.
**Purpose**: Allow adjustments to closed periods without reopening
**Usage**: Audit adjustments, prior period corrections
**Posting Date**: Shows original transaction month
**Adjustment Date**: Shows actual posting month

**Configuration**:
**Navigation**: Setup > Accounting > Set Up Adjusting Periods

**Settings**:
- Create adjusting period for each fiscal period
- Adjusting period remains open after main period closes
- Adjusting period closes with next month
- Example: "Adj Jan 2026" closes when Feb 2026 closes

**Business Rules**:
- All adjusting entries require approval
- Adjusting entries documented with reason
- Materiality threshold: $5,000 (requires CFO approval)
- Adjusting entries reported separately

## Period Permissions

### Role-Based Access

**Controller Role**:
- Open/Close periods
- Post to any open period
- Reopen closed periods (with approval)
- Post adjusting entries
- Override period lock (emergency only)

**Accountant Role**:
- Post to open periods
- View closed periods
- Request period reopen
- Post adjusting entries (with approval)

**Store Manager / Standard User**:
- Post to current period only
- Cannot post to closed periods
- Cannot post adjusting entries
- Read-only access to prior periods

**Executive Role (CEO/CFO)**:
- View all periods
- Approve period reopen requests
- View all reports across periods

## Business Rules

### Transaction Dating Rules
1. Transactions post to period based on transaction date
2. No future dating beyond current open period
3. Backdating requires Controller approval
4. System date vs. transaction date validation

### Period Lock Rules
1. Periods lock automatically after close (can be configured)
2. Reopen requires CFO approval via workflow
3. Reopen period: 5-day maximum
4. Reason for reopen documented
5. All changes in reopened period tracked

### Accrual Rules
1. Accruals posted last day of period
2. Reversing entries post first day of next period
3. Accrual accounts monitored for balance
4. Quarterly accrual review
